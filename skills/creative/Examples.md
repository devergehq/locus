# Creative — Examples

Short illustrations of creative work in different modes.

## Example 1 — Copy: an error message

**Task:** Error message for "this email is already registered."

**Generic output (weak):**
> An account with this email address already exists. Please sign in or use a different email address.

Functional. Empty. Exists in a million apps.

**Specific output (stronger):**
> You've been here before. Sign in with this email, or reset the password if you've forgotten it.

Same function. Different tone. The specificity — "You've been here before" — does the work. It names what happened in human terms instead of describing a system state.

**Constraint that produced the improvement:** 15 words max; no use of "please" or "address"; must acknowledge prior visit, not just assert duplicate.

## Example 2 — Visual: a single-card visualisation

**Task:** Visualise "search result" for a site with heavy content.

**Generic output (weak):**
A card with a title, description, and URL. Every search engine on earth.

**Specific output (stronger):**
A card with the title, the most relevant 40-character snippet (with the query term highlighted), and three tiny icons — one per signal that makes this result trustworthy (recent date, authoritative domain, match depth). The description field drops entirely.

**Constraint that produced it:** no more than five information elements; every element must help the user decide whether to click, not just describe the result.

## Example 3 — Name: a product name

**Task:** Name for an agentic workflow framework.

**Generic approaches (weak):**
- Technical: "WorkflowKit", "AgentFlow"
- Marketing: "Synergy", "Apex"
- Abbreviation: "WAF" (workflow-as-framework)

All forgettable. All generic.

**Specific (stronger):** "Locus"

The name does work:
- Evokes *place* — a locus of control, focus, attention.
- Short enough to type as a CLI name.
- Has no existing product collision in the AI tooling space.
- Reads the same in every language.
- Sounds confident without being bombastic.

**Constraint that produced it:** 4-6 letters; must work as both a CLI command and a spoken noun; must not be a cute portmanteau.

## Example 4 — Structure: reorganising an essay

**Task:** Essay argument that "the answer is in a weird spot."

**Generic:** Intro → claim → three supporting points → conclusion. Standard five-paragraph form.

**Specific:** Open with the weird spot itself — the observation that doesn't fit — as the first sentence. Spend the first third of the essay sitting in the anomaly. Explicitly resist resolving it. Then, having earned the reader's attention by the anomaly, move to the explanation.

**Constraint that produced it:** the reader should be uncertain until paragraph four; the opening must not telegraph the conclusion.

## Example 5 — Negative space: what was left out

**Task:** Brand palette for a serious B2B tool.

**Generic (weak):** Logo + primary colour + accent + neutral. Apply everywhere.

**Specific (stronger):** Logo + a single colour + generous use of white. No accent colour; no secondary system. The restraint itself is the brand signal — competitors fill space with colour to signal sophistication, so emptiness becomes distinctive.

**Constraint that produced it:** one colour only; must feel more serious than 80% of the category.

---

These examples share a pattern:

1. Name the generic path first.
2. Impose one or two deliberate constraints.
3. The constraint forces specific output.
4. Verify the output against the goal, not against the method.
